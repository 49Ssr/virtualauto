# VirtualAuto Project Doctrine

## Source confidence

Every imported idea should be classified:

- Observed: directly seen or measured
- Sourced: backed by documentation or literature
- Hypothesis: plausible but unverified
- Implemented: built in Blender or tooling
- Validated: tested against evidence

## Asset archaeology

Game assets should be treated as production data, not simple meshes.

Preserve:

- provenance
- unknown attributes
- resource relationships
- conversion history
- validation results

## Implementation standard

Blender systems should document:

- version compatibility
- node types
- inputs and outputs
- coordinate spaces
- limitations
- performance impact
- validation method

## Experiments

Prefer small controlled experiments:

one phenomenon
one test asset
one hypothesis
one validation method
