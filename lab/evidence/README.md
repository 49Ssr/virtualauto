# Evidence

This directory stores small, lawful evidence records and manifests. Large
images, EXRs, videos, captures, and extracted source assets belong in approved
external storage; commit only their checksums, provenance, and stable references
here. Small original `.blend` regression fixtures have a narrow Git LFS
exception defined in [the binary asset policy](../governance/BINARY_ASSET_POLICY.md).

Evidence does not become a conclusion automatically. Keep direct observation,
interpretation, and decision as separate fields.

Exploratory look-development is not a regulated transaction. Changing a colour,
roughness, light, or procedural control in a disposable working scene does not
require an evidence, observation, and claim record. Create records when a result
is retained as a comparison, supports a decision, qualifies an asset, or is
promoted into a reusable implementation. This keeps experimentation fast while
making accepted checkpoints reproducible.

## Low-friction capture

For a retained artifact already under `lab/evidence/`, let VirtualAuto calculate
the checksum, byte size, timestamp, schema link, and repository path:

```text
virtualauto record-evidence lab/evidence/runs/EXP-EXAMPLE-001/result.txt \
  --id EVD-EXAMPLE-RESULT-001 \
  --experiment-id EXP-EXAMPLE-001 \
  --artifact-type log \
  --provenance "Generated by the documented experiment procedure." \
  --rights-status original
```

The command deliberately does not manufacture an observation or claim. Those
require human or instrument interpretation and must remain explicit. Private
external assets use `virtualauto register-source` and an opaque storage
reference instead of leaking a filesystem path.
